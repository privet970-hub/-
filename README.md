# **오픈소스 과제**

이 문서는 리눅스 시스템을 제어하는 데 사용되는 네 가지 핵심 명령어 top, ps, jobs, kill에 대한 보고서입니다.

**목차**

1. top (시스템 모니터링)
2. ps (프로세스 스냅샷)
3. jobs (쉘 작업 관리)
4. kill (프로세스 시그널 전송)

# **1. top (시스템 모니터링)**
>top 명령어는 시스템의 현재 상태를 **실시간(Dynamic)**으로 동적으로 보여주는 도구입니다. 서버의 성능 문제를 진단하거나 부하를 모니터링하는 데 가장 흔하게 사용됩니다.

**주요 정보**<br>
+ 시스템 부하 (Load Average): 1분, 5분, 15분 동안의 평균 시스템 부하 수준.<br>
+ CPU 통계: User, System, Idle 등 CPU 사용률.<br>
+ 메모리/Swap: 현재 사용 가능한 메모리와 스왑(Swap) 공간 현황.<br>
+ 프로세스 목록: PID, 사용자, CPU 사용률(%CPU), 메모리 사용량(%MEM) 등을 포함한 프로세스 목록.<br>

**사용 예시**<br>
>기본 실행 (실시간 모니터링 시작)<br>
top

`top` 실행 중 유용한 키

| 키 | 설명 |
| :---: | :-----------------------: |
| q | top 명령어 종료 |
| k | 선택된 프로세스에 kill 신호를 보냄 |
| P | CPU 사용량 순으로 정렬 (기본값) |
| M | 메모리 사용량 순으로 정렬 |

# **2. ps (Process Status: 프로세스 스냅샷)**<br>
>ps 명령어는 명령어를 실행한 특정 시점의 실행 중인 프로세스 정보를 스냅샷 형태로 출력합니다.

**주요 옵션**<br>
| 옵션 | 설명 |
| :----: | :--------------------------------------------- |
| a | 터미널과 연결된 모든 프로세스 표시 |
| u | 사용자(User) 기반 형식으로 상세 정보 표시 |
| x | 터미널에 연결되지 않은 프로세스도 표시 |
| -e | 모든 프로세스 표시 (Every process) |
| -f | 전체 형식 (Full format)으로 상세 정보 표시 (UID, PPID 등) |

**사용 예시**<br>
>가장 일반적인 사용법: 모든 프로세스를 상세하게 출력<br>
ps aux<br><br>
모든 프로세스를 전체 형식으로 출력하고, 파이프를 통해 'nginx'가 포함된 프로세스만 검색<br>
ps -ef | grep nginx<br>


# **3. jobs (쉘 작업 관리)**<br>
>jobs 명령어는 현재 로그인된 쉘 세션 내에서 백그라운드나 정지 상태로 실행 중인 명령어, 즉 **작업(Job)**의 목록을 보여줍니다.

**주요 기능**<br>
+ 백그라운드로 실행한 작업 ($ command &) 확인
+ Ctrl + Z로 일시 정지시킨 작업 확인

**사용 예시 및 제어 명령어**<br>
> #현재 쉘의 작업 목록 확인<br>
jobs<br><br>
#예시: 백그라운드 작업 실행<br>
$ sleep 60 &<br>
[1] 12345<br><br>
#작업 번호 1을 포그라운드로 가져옴<br>
fg %1<br><br>
#작업 번호 1을 백그라운드로 다시 실행<br>
bg %1

+ [1]은 **작업 번호(Job Number)**이며, 뒤의 12345는 **프로세스 ID (PID)**입니다.

# **4. kill (프로세스 시그널 전송)**<br>
>kill 명령어는 지정된 **프로세스 ID (PID)**에게 특정 **시그널(Signal)**을 전송하여 프로세스를 제어합니다.<br><br>

**주요 시그널**<br>
| 시그널 번호 | 시그널 이름 | 설명 |
| :--------: | :--------: | :----- | 
| 15 | SIGTERM | 정상 종료 요청 (Graceful Shutdown) - 기본 시그널 |
| 9 | SIGKILL | 강제 즉시 종료 (Immediate Termination) - 프로세스가 정리할 시간 없음 |
| 1 | SIGHUP  | 설정 파일 재로드 또는 재시작 요청 |

**사용 예시**<br>
>#12345 PID 프로세스에게 정상 종료(SIGTERM) 요청 (기본값)<br>
kill 12345<br><br>
#54321 PID 프로세스를 강제 종료(SIGKILL)<br>
kill -9 54321<br><br>
#작업 번호 2에게 정상 종료 요청 (jobs로 확인된 작업)<br>
kill %2<br><br>
